# by `mosquitto_passwd`. This file is a placeholder so the broker starts
# in dev; populate real hashes with:
#
#   docker run --rm -v $PWD:/m eclipse-mosquitto:2.0 \
#     mosquitto_passwd -b /m/passwords.txt <user> <pw>
#
# Required dev users (one row per line, format `user:hash`):
#   station-4         (PLC test device, can publish fab/munich/plc/station-4)
#   camera-12         (inference test device, fab/munich/inference/camera-12)
#   event-ingestion   (subscriber service account, read fab/+/+/+)
#
# Until populated the broker will refuse all connections (allow_anonymous
# false). The MQTT subscriber hosted service (spec 006 PR D) requires
# this file to be populated before it can connect.
